图的存储

选择哪种储存方式主要看主要操作

  • 遍历出度
  • 遍历入度
  • 静态/动态
  • 查询某个边是否存在
  • 等待补充

~邻接矩阵

实现：

  • 用一个二维数组，数组下标表示节点的序号，值表示边权
  • 可以考虑离散化节点序号

复杂度：

  • 查询是否存在某条边：O(1)。

    遍历一个点的所有出边：O(N)。

    遍历整张图：O(N^2)。// 不连续

    空间复杂度：O(N^2)。

适用：

  • 邻接矩阵只适用于没有重边（或重边可以忽略）的情况。

  • 其最显著的优点是可以 查询一条边是否存在。

  • 由于邻接矩阵在稀疏图上效率很低（尤其是在点数较多的图上，空间无法承受），所以一般只会在稠密图上使用邻接矩阵。

  • 出度、入度都可O(E)的查询

~邻接表:高空间利用率邻接矩阵

实现：

  • 使用一个支持动态增加元素的数据结构构成的数组(vector / list)，如 vector<int> adj[n + 1] 来存边，其中 adj[u] 存储的是点u 的所有出边的相关信息（终点、边权等）。
  • 用链表的话方便删除

复杂度：

  • 查询是否存在u到v的边：O(d(u))（如果事先进行了排序就可以使用 二分查找 做到O(log(d(u)))）。

    遍历点u的所有出边：O(d(u))。

    遍历整张图：O(n+m)。// 连续

    空间复杂度：O(n+m)。

适用：

  • 存各种图都很适合，除非有特殊需求（如需要快速查询一条边是否存在，且点数较少，可以使用邻接矩阵）。

  • 尤其适用于需要对一个点的所有出边进行排序的场合。

~链式前向星

ps：优点是边是带编号的，有时会非常有用，而且如果 cnt 的初始值为0，存双向边时 i ^ 1 即是 i 的反边（常用于 网络流）。

奇数 ^ 1 -=1;

偶数 ^ 1 += 1;

  • 代码实现

    void addEdge(int u, int v){
  	to[cnt] = v;
  	next[cnt] = head[u];
  	head[u] = cnt++;
}
// 遍历u的出边
for(int i = head[u];~i;i = next[i]){
  	int v = to[i];
  	// v 为出边所指向的点。
}

图的遍历

~与树的区别：

  • 图可能有回路：记录标记数组，防止走循环死路
  • 图不一定连通：从一个节点出发不一定达到所有节点
    • 每次都从一个没有被访问的节点开始dfs
    • 然后访问出边到达的没有被访问过的节点

～DFS：

  • DFS 最显著的特征在于其递归调用自身。同时与 BFS 类似，DFS 会对其访问过的点打上访问标记，在遍历图时跳过已打过标记的点，以确保每个点仅访问一次。符合以上两条规则的函数，便是广义上的 DFS。

  • 代码实现：

    void dfs(int u) {
  vis[u] = 1;
  for (int i = head[u];~i; i = edge[i].nxt) {
    if (!vis[edge[i].to]) {
      dfs(v);
    }
  }
}

  • DFS 序列是指 DFS 调用过程中访问的节点编号的序列。每个子树都对应 DFS 序列中的连续一段（一段区间）。（线段树）

  • 在 DFS 过程中，通过记录每个节点从哪个点访问而来，可以建立一个树结构，称为 DFS 树。DFS 树是原图的一个生成树。

    DFS树有很多性质，比如可以用来求强连通分量。

  • DFS在使用当中往往外面套了层壳来保证每个点都能被遍历到

    void visit{
 		for(int i = 1;i < n;++i){
      	if(vis[i] == 1) continue;
     		dfs(i);
    } 
}

～BFS：

  • 所谓宽度优先。就是每次都尝试访问同一层的节点。 如果同一层都访问完了，再访问下一层。这样做的结果是，BFS 算法找到的路径是从起点开始的最短合法路径。换言之，这条路所包含的边数最小(非边权和最小)。

  • 代码实现

    void bfs(int u) {
  while (!Q.empty()) Q.pop();
  Q.push(u);
  vis[u] = 1;//标记数组，表示这个点是否已经被搜索过
  d[u] = 0; //到起点的深度
  p[u] = -1; //记录从起点到这个点的最短路上的上一个点。
  while (!Q.empty()) {
    u = Q.front();
    Q.pop();
    for (int i = head[u]; i; i = e[i].x) {//链式前向星
      if (!vis[e[i].t]) {
        Q.push(e[i].t);
        vis[e[i].t] = 1;
        d[e[i].t] = d[u] + 1;
        p[e[i].t] = u;
      }
    }
  }
}
void restore(int x) { //恢复最短路径上的点
  vector<int> res;
  for (int v = x; v != -1; v = p[v]) {
    res.push_back(v);
  }
  std::reverse(res.begin(), res.end());
  for (int i = 0; i < res.size(); ++i) printf("%d", res[i]);
  puts("");
}

  • 应用：

    • 在一个无权图上求从起点到其他所有点的最短路径。(有权图有其他的算法)
    • 在O(N+M)时间内求出所有连通块。(我们只需要从每个没有被访问过的节点开始做 BFS，显然每次 BFS 会走完一个连通块）
    • 如果把一个游戏的动作看做是状态图上的一条边（一个转移），那么 BFS 可以用来找到在游戏中从一个状态到达另一个状态所需要的最小步骤。
    • 在一个边权为0/1的图上求最短路。(需要修改入队的过程，如果某条边权值为 0，且可以减小边的终点到图的起点的距离，那么把边的起点加到队列首而不是队列尾）
    • 在一个有向无权图中找最小环。（从每个点开始 BFS，在我们即将抵达一个之前访问过的点开始的时候，就知道遇到了一个环。图的最小环是每次 BFS 得到的最小环的平均值。）
    • 找到一定在(a,b)最短路上的边。分别从 a 和 b 进行 BFS，得到两个 d 数组。之后对每一条边(u,v)，如果da[u]+1+db[v] == da[b]，则说明该边在最短路上
    • 找到一定在(a,b)最短路上的点。分别从a和b进行 BFS，得到两个d数组。之后对每一个点 v，如果da[v]+db[v] == da[b] ，则说明该点在最短路上。

  • 例子：

    • 三个自由度的bfs. oj1195

～双端队列BFS：

  • 适用范围：边权值为可能有，也可能没有（由于 BFS 适用于权值为 1 的图，所以一般是 0 or 1），或者能够转化为这种边权值的最短路问题。

    例子：例如在走迷宫问题中，你可以花 1 个金币走 5 步，也可以不花金币走 1 步，这就可以用 0-1 BFS 解决。

  • 实现：一般情况下，我们把(没有权值的边)扩展到的点放到队首，(有权值的边)扩展到的点放到队尾。这样即可保证在整个队列中，像普通 BFS 一样，越靠近队首，权值越小，且权值零一之间有分隔。

  • 代码实现：

    while (队列不为空) {
  int u = 队首;
  弹出队首;
  for (枚举 u 的邻居) {
    更新数据
    if (...)
      添加到队首;
    else
      添加到队尾;
  }
}

最小生成树

~定义：

  • 生成树：一个连通无向图的生成子图，同时要求是树。也即在图的边集中选择n-1条，将所有顶点连通，形成一棵树。

    只有连通图才有生成树

  • 最小生成树为：边权和最小的生成树

~性质：

  • MST的最大边权是这个图所有生成树的最大边权中最小的。

~Kruskal 算法

算法的基本思想是从小到大加入边，采用贪心的思想。

原理：

将边排序（键值从小到大） 贪心

也可以采用优先队列 可能常数更优秀

然后从小到大遍历边(u,v)。当u、v已经被连在一起时，跳过这条边。

点是否被连在一起采用并查集进行维护。

代码实现：

void merge_sort(T * data,int l,int r);
void merge(int x,int y);
void check(int x,int y);
int find(int x);

// main precedure
{
  	merge_sort(Edge,1,size);
  	for(int i = 1;i <= edgeSize;++i){
      	if(check(Edge[i].x,Edge[i].y)) continue;
      	merge(Edge[i].x,Edge[i].y);
      	ans += 1;
      	length += Edge[i].weight;
      	if(ans == edgeSize - 1) return;
     		//若不能return说明没有最小生成树
    }
}

判断MST是否唯一：

做两次Kruskal，将第一次被放入最小生成树的边打上标记，第二次优先处理没有打标记的。若能成功，则说明不唯一。

~Prim算法

原理：

每次加点，同样是贪心的思想。加入的点是每次选点时离生成树边权最小的点。

这样选保证了这个点通过这条边加入生成树是最优的 类似于dijkstra

所以要维护每个点离生成树的边权最小值

最后边权和就是 low_cost[i] 1<=i<=n求和

代码实现：

void prim(){
  	priority_queue<Edge> que;
  	while(!que.empty()) que.pop();
  	int low_cost[nodeSize]; //如果节点到生成树的距离
  	int visit[nodeSize] = {0};
  	for(int i = 1;i <= nodeSize;++i) low_cost[i] = 0x7f7f7f7f;
  	for(int i = head[1];~i;i = edge[i].nxt){
      	que.push({1,edge[i].to,edge[i].weight});
      	low_cost[edge[i].to] = edge[i].weight;
    }
    visit[1] = 1;low_cost[1] = 0;
  	for(int i = 1;i < nodeSize;++i){
      	Edge tmp = que.top();que.pop();
      	visit[tmp.to] = 1;
      	for(int i = head[tmp.to];~i;i=edge[i].nxt){
          	if(visit[edge[i].to]) continue;
          	if(edge[i].weight < low_cost[edge[i].to]){
              	low_cost[edge[i].to] = edge[i].weight;
              	que.push({tmp.to,edge[i].to,edge[i].weight});
            }
        }
    }
}

~Boruvka算法

拓扑排序

拓扑排序要解决的问题是给一个图的所有节点排序，使得前面的节点不“依赖于”后面的节点。

因此我们可以说在一个有向无环图中，将图中的顶点以线性方式进行排序，使得对于任何的顶点u到v的有向边(u,v) , 都可以有u在v的前面。

~DFS算法

原理：

DFS过程中记录每个节点的访问时间和回溯时间，当$u$回溯时间小于$v$时，不存在从$u\rightarrow v$的有向边。

proof: 反证法，tree edge(显然有$t(v)<t(u)$矛盾)；forward edge(说明$v$在$u$的子树上，$v$先回溯，矛盾)；cross edge(说明$v$先被访问，否则$u,v$会在同一棵子树上，矛盾)；back edge(与大前提DAG矛盾)。

所以说每个图当中回溯时间最小的节点一定是tail(出度为零)，一定是当前图拓扑排序的最后一位，所以回溯时将当前节点压入栈中即可。

时间复杂度：

O(E+V)

代码实现：

void dfs(int u){
  	if(visit[u]) return;
  	for(int i = head[u];~i;i = next[i]){
     		int v = to[i];
      	if(visit[v]) continue;
      	visit[v] = true;
      	dfs(v);
    }
  	ans.push(u);	// 回溯最早的节点压入栈中 最后逆序输出
}

~kahn算法

原理：

初始状态下，集合S装着所有入度为0的点， L是一个空列表。

每次从S中取出一个点u（可以随便取）放入L , 然后将u的所有边删除。对于边(u,v)，若将该边删除后点v的入度变为0，则将v放入S中。

不断重复以上过程，直到集合S为空。检查图中是否存在任何边，如果有，那么这个图一定有环路(其中节点不会进入集合S中)，否则返回L ， L中顶点的顺序就是拓扑排序的结果。

当然从出度为零的点开始做也是可以的。

时间复杂度：

O(E+V)

代码实现：

bool toposort() {
  	q = new queue();
  	for (i = 0; i < n; i++){
    		if (in_deg[i] == 0) q.push(i);
  	}
  	ans = new vector();
  	while (!q.empty()) {
    		u = q.pop();
    		ans.push_back(u);
    		for each edge(u, v) {
      			if (--in_deg[v] == 0) q.push(v);
    		}
  	}
  	if (ans.size() == n) {
    		for (i = 0; i < n; i++)
      			std::cout << ans[i] << std::endl;
    		return true;
  	} else {
    		return false;
  	}
}

字典序tupo排序：

将S采用的队列替换为优先队列即可。

关键路径：

AOE网络的应用

  • 顶点表示事件，有向边的权值表示某个活动的持续时间，有向边的方向表示活动和事件的先后次序。

    进入顶点的所有活动都结束后，顶点表示的事件才可以发生，从该顶点出发的所有活动才可以开始。

  • 入度为0：开工事件；出度为0:竣工事件

  • 完成整项工程至少需要多少时间：源点到收点的最长路径的长度，称为关键路径

  • 算法：

    • 最早发生时间：事件的最早发生时间

      每个直接前驱的最早发生时间加上从该前驱到该顶点的活动时间的最大者

    • 最迟发生时间：不影响整个项目按时完成的前提下，这个事件最晚可以什么时候发生

      每个直接后继的最迟发生时间减去顶点到该直接后继的活动时间的最小者就是该顶点的最迟发生时间。

    • 最早发生时间和最晚发生时间相同的顶点序列构成关键路径

  • 具体实现：

    先用拓扑排序得到活动发生的顺序。这样保证拓扑序

    按拓扑序顺序可得到每个事件的最早发生时间

    逆序可得到每个事件的最迟发生时间

    最后比较即可得到关键事件

    然后按拓扑序if输出

应用：

判断有向图是否存在环路

判断无向图是否存在环路：

  1. 类似于有向图环路拓扑法：

    将所有度数<=1的节点删除并删除对应边

    将所有度数为1的节点放入队列中开始删除 每次删除后将新的度数为1的点加入队列

    看最后有没有剩下节点

  2. DFS/BFS：

    搜索的本质就是形成一棵搜索树

    如果某一个节点被访问了两次说明出现了回边。

最短路径问题

~输出最短路

开一个 pre 数组，在更新距离的时候记录下来后面的点是如何转移过去的，算法结束前再递归地输出路径即可。

～Floyd算法

for(int i = 1;i <= n;++i)
  for(int j = 1;j <= n;++j) f[i][j] = 0x7f7f7f7f;
for(int i = 1;i <= edgeSize;++i) f[edge[i].from][edge[i].to] = edge[i].weight;
for(int i = 1;i <= n;++i) f[i][i] = 0;
for (k = 1; k <= n; k++) {
    for (i = 1; i <= n; i++) {
        for (j = 1; j <= n; j++) {
            f[i][j] = min(f[i][j], f[i][k] + f[k][j]);
        }
    }
}

~SPFA算法

原理：

dist[x] 表示节点x到源的当前最短路径

  • 松弛操作: 通过 dist[y] + edge(y,x) 更新dist ，因为最短路一定比其他路径要短
  • 初始化时所有都设置为0x7f7f7f7f(INT_MAX)。从起点开始松弛周围的点，被松弛的点才可能去改变其他点到起点的最短路，所以加入队列。直到被松弛过的队列为空。

代码实现：

void SPFA(int start){
  	for(int i = 1;i <= nodeSize;++i) dist[i] = 0x7f7f7f7f;
  	memset(visit,0,sizeof(visit));
  	dist[start] = 0;
  	visit[start] = 1;
  	while(!que.empty) que.pop();
  	que.push(start);
  	while(!que.empty){
      	int now = que.front();que.pop();
      	for(int i = head[now];~i;i = edge[i].next){
          	int v = edge[i].to;
          	if(dist[v] > dist[now] + edge[i].weight){ // relax
              	dist[v] = dist[now] + edge[i].weight;
              	if(!visit[v]){
                  	que.push(v);
                  	visit[v] = 1;
                }
            }
        }
      	visit[now] = 0;
    }
}

复杂度：

虽然在大多数情况下 SPFA 跑得很快，但其最坏情况下的时间复杂度为O(MN)，将其卡到这个复杂度也是不难的

~dijkstra算法

只适用于非负权图，但是时间复杂度非常优秀。

原理：

对于一个有向图或无向图，所有边权为正，给定a和b，求a到b的最短路，保证a一定能够到达b。这条最短路是否一定存在呢？答案是肯定的。相反，最长路就不一定了，由于边权为正，如果遇到有环的时候，可以一直在这个环上走，因为要找最长的，这样就使得路径越变越长，永无止境，所以对于正权图，在可达的情况下最短路一定存在，最长路则不一定存在。这里先讨论正权图的最短路问题。

经过tarjan缩点后 无环。最长路和最短路都可以用最短路算法求解

最短路满足最优子结构性质，是一个动态规划问题。

最短路的最优子结构可以描述为：D(s, t) = {Vs … Vi … Vj … Vt}表示s到t的最短路，其中i和j是这条路径上的两个中间结点，那么D(i, j)必定是i到j的最短路，这个性质是显然的，可以用反证法证明。

基于上面的最优子结构性质，如果存在这样一条最短路D(s, t) = {Vs … Vi Vt}，其中i和t是最短路上相邻的点，那么D(s, i) = {Vs … Vi} 必定是s到i的最短路。Dijkstra算法就是基于这样一个性质，通过最短路径长度递增，逐渐生成最短路。

如果源点到x点的最短路已经求出，并且保存在d[x]，则可以利用x去更新x能够直接到达的点y的最短路。

当前所有被松弛过的点中dist值最小的即被求出最短路径(不可能再被更新)，然后用这个点再去松弛其他的点。

代码实现:

struct node{
  	int index,dist;
  	bool operator < (const node & o) const{}
}
for(int i = 1;i <= nodeSize;++i){
  	dist[i] = 0x7f7f7f7f;
}
dist[start] = 0;
q.push(node{start,0});
while(!q.empty()){
		node x = q.top();
		q.pop();
		int u = x.index;
		if(vis[u]) continue; // 已经被确定最短路的标记；而且这时候如果还在队列中说明这个dist已经过时了。
		else vis[u]=1;
  	for(int i=head[u];~i;i=edge[i].next){
				if(dis[edge[i].to] > dis[u] + edge[i].len){
						dis[edge[i].to] = dis[u]+edge[i].len;
						q.push(node{edge[i].to,dis[edge[i].to]});
				}
		}
}

强连通分量

～定义：

  • 在有向图G中，如果任意两个不同的顶点相互可达，并且maximal，则称该有向子图是强连通的。

    Strong Connect Components

  • DAG：有向无环图(Directed Acyclic Graph)

    图存在拓扑排序等价于图是DAG

    • 有向无环图中唯一出度为0的点,一定可以由任何点出发均可达(由于无环,所以从任何点出发往前走,必然终止于一个出度为0的点)

    • 有向无环图中所有入度不为0的点，一定可以由某个入度为0的点出发可达。 (由于无环，所以从任何入度不为0的点往回走，必然终止于一个入度为0的点)。

      所以入度为零的所有点使得整个图能够被遍历完全

  • 经过强连通分量缩点处理后，新图像一定是DAG。可以利用上面的性质。

  • 强连通分量完全分隔一个图，没有遗漏也没有重叠。

～应用：

我们可以将一张图的每个强连通分量都缩成一个点。

  • 那么重新建图后新图为DAG
  • 可以进行拓扑排序\最小生成树\最短路径等多种算法

～tarjan算法：

原理：

  • 做一遍DFS，用dfn[i]表示编号为i的节点在DFS过程中的访问序号(也可以叫做开始时间)。在DFS过程中会形成一搜索树。在搜索树上越先遍历到的节点，显然dfn的值就越小。dfn值越小的节点，就称为越“早” 。用low[i]表示从i节点出发DFS过程中i下方节点(开始时间大于dfn[i]，且由i可达的节点)所能到达的最早的节点的开始时间。初始时low[i]=dfn[i]。

  • DFS过程中，碰到哪个节点，就将哪个节点入栈。栈中节点只有在其所属的强连通分量已经全部求出时，才会出栈。

  • 如果发现某节点u有边连到栈里的节点v(返祖边)，则更新u的low值为min(low[u],dfn[v]) ，若low[u]被更新为dfn[v],则表明目前发现u可达的最早的节点是v；对于u的子节点v，从v出发进行的DFS结束回到u时，使得 low[u] = min(low[u],low[v])。**因为u可达v, 所以v可达的最早的节点，也是u可达的；**如果一个节点u，从其出发进行的DFS已经全部完成并回到u，而且此时其low值等于dfn值，则说明u可达的所有节点，都不能到达任何比u早的节点。因此该节点u就是一个强连通分量在DFS搜索树中的根；此时，显然栈中u上方的节点，都是不能到达比u早的节点的。将栈中节点弹出，一直弹到u(包括u), 弹出的节点就构成了一个强连通分量。

代码实现：

void tarjan(int a){
    dfn[a] = low[a] = ++timeclock;
    instack[a] = 1;
    stack[++top] = a;
    for(int i = head[a]; ~i ;i = Edge[i].next)
    {
        int v = Edge[i].to;
        if(!dfn[v]) //没有访问过的
        {
            tarjan(v);
            low[a] = min(low[a],low[v]);//它能到达所连结点能到达的最早结点
        }
        else if(instack[v]) { // 被访问过且在栈中
            low[a] = min(low[a], dfn[v]); // 说明这是一条返祖边
        }
    }
    if(dfn[a] == low[a]) //自己包括自己所能到的点所能到达的最早的点是自己 说明自己是一个根结点
    {
        tot++;// 缩点后新图的节点个数
        int v = 0;
        while(a != v){
            v = stack[top--];
            color[v] = tot;
            sum[tot] += money[v];//权值收缩
            instack[v]=0;
        }
    }
}

//新建图
memset(head,-1,sizeof(head));
memset(Edge,0,sizeof(Edge));
tmpcount = 0;
for(int i = 1;i <= edgeSize;++i){
    if(color[from[i]] != color[to[i]]) {
        addEdge(color[from[i]],color[to[i]],sum[color[to[i]]]);
    }
}

~DFS算法

原理：

利用缩点后的图是DAG的性质，其tail scc当中的点能访问到的点均属于本scc。问题转化为如何快速找到当前图中的一个点使得该点是该图缩点后tail scc当中的一个点。

Lemma ： The SCC contains the largest finish time vertex must be the head SCC.

proof: 反证法。假设$u$的回溯时间最大，且存在来自其他scc的$v:v\rightarrow u$。显然，$v$不能比$u$早被访问，否则$u$必然是$v$的dfs子树，回溯时间小于$v$。其次，$v$不能在其他的dfs树当中，否则$v$必然比$u$开始晚，结束也晚。所以说，$u,v$必须在同一棵dfs树上且要先访问$u$，说明存在$u\rightarrow v$ ，则$u,v$属于同一个scc，产生矛盾。

所以取$G$的反图$G^R$，再dfs遍历找到回溯最晚的点$u$，即可找到$G$当中属于tail scc的一个点$u$。

伪代码：

  1. Construct $G^R$ and dfs it. Maintain a sorted list by the finish time.
  2. DFS $G$ by the descending order of the finish time(Do not start from a visited vertex).
  3. Each visit forms a SCC.

欧拉图

~定义：

  • 通过图中所有边恰好一次且行遍所有顶点的通路称为欧拉通路。

  • 通过图中所有边恰好一次且行遍所有顶点的回路称为欧拉回路。(要回去)

  • 具有欧拉回路的无向图或有向图称为欧拉图。

  • 具有欧拉通路但不具有欧拉回路的无向图或有向图称为半欧拉图。

非形式化地讲，欧拉图就是从任意一个点开始都可以一笔画完整个图，半欧拉图必须从某个点开始才能一笔画完整个图。

~性质：

  • 欧拉图中所有顶点的度数都是偶数。

  • 若G是欧拉图，则它为若干个边不重的圈的并。

  • 若G是半欧拉图，则它为若干个边不重的圈和一条简单路径的并。

~判别：

  • 对于无向图 G， G是欧拉图当且仅当G是连通的且G没有奇度顶点。

  • 对于无向图 G， G是半欧拉图当且仅当G是连通的且G中恰有0个或2个奇度顶点(一个起点 一个终点)。

  • 对于有向图 G， G是欧拉图当且仅当G的所有顶点属于同一个强连通分量且每个顶点的入度和出度相同。

  • 对于有向图 G， G是半欧拉图当且仅当G有n-2个顶点的入度==出度，1个顶点+1(起点)，1个顶点-1(终点)
